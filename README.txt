Автотестирование выполнено на PyTest с использованием Selenium WebDriver.

Список файлов:
requirements.txt 	- файл с описанием тестового окружения
test_apps_page.py 	- содержит набор тест-кейсов
apps_page.py 		- содержит описание необходимых процедур/функций в используемых классах.
base_page.py 		- содержит абстрактный класс.
locators.py 		- содержит локаторы нужных элементов на Web страницах
conftest.py 		- специальный файл, в котором описаны возможности WebDriver по подключению к Web странице разными браузерами (chrome, firefox) и с разными языковыми настройками.


Автотест прогонялся в виртуальном окружении
Командой "pip install -r requirements.txt"  можно развернуть у себя эквивалентное окружение.


Возможные команды запуска автотеста:
pytest -s -v test_apps_page.py
pytest -s -v --browser_name=firefox  --language=en  test_apps_page.py
краткая запись логов:
pytest -v --tb=line test_apps_page.py
pytest -v --tb=line --browser_name=firefox --language=en test_apps_page.py


Описание проверок:

Страница https://www.cft.ru/apps#_tab1 представляет набор именованных кнопок с описанием. 

Для проверки по каждой кнопке использовался следующий алгоритм:
1) Открываем базовую страницу https://www.cft.ru/apps
2) Проверяем наличие кнопки и ее наименование
3) Проверяем текст в информационных полях относящихся к тестируемой кнопке.
4) Кликаем по тестируемой кнопке, далее возможны два варианта а) активное окно загружает новую страницу, б) Открывается новое окно с новой страницей.
5) Проверяем URL на новой странице и сравниваем с ожидаемым.
